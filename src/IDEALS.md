# IDEALS - Design Improvements Found During Development

This file documents improvements and ideals discovered while analyzing existing patterns and implementing components. These should be applied immediately if they're improvements (non-breaking).

## General Principles

- **Standard Sizing**: All components (buttons, badges, selects, inputs, etc.) should follow this sizing convention:
  - `mini`: 32px height
  - `sm`: 36px height (not 32px)
  - `md`: 40px height (default)
  - `lg`: 48px height

## Component-Specific Improvements

### Buttons

- ✅ Added `mini` size (32px) - smallest size for compact UIs
- ✅ Updated `sm` size from 32px to 36px - standard small size
- ✅ Applied consistent sizing across Button, IconButton components
